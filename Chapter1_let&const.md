# let 和 const命令

## let 
> 代码块内有效  
> 不存在变量提升, 声明后使用  
> 不能重复声明

Q: 为什么需要块级作用域？  
A: 没有块级作用域带来得问题： 1、内层变量覆盖外层变量 2 泄露计数器为全局变量

ES6规定，块级作用域之中，函数声明语句得行为类似于let，在块级作用域之外不可引用

为了兼容老代码，ES6在附录B中规定，浏览器的实现可以不遵守以上规定。

* 允许在块级作用域内声明函数。
* 函数声明类似于var，即会提升到全局作用域或函数作用域的头部。
* 同时，函数声明还会提升到所在的块级作用域的头部。

## const

常量，一旦声明，不能修改。  
对于 const obj = {};  来说，指向的是存储地址
